Glossary
========

| Term                        | Definition                                                                                                                                                                                                         |
|-----------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **ADR**                     | Architecture Decision Record — a short document capturing a significant architectural decision, the context in which it was made, and its rationale. Stored in `docs/adr/`.                                         |
| **Audit Log**               | A record of API server access and operations. KIM maps each cluster's tenant ID to an external audit log backend via the converter configuration. Audit logging is mandatory by default (`--audit-log-mandatory`).   |
| **BTP**                     | SAP Business Technology Platform — the cloud platform on which Kyma is offered as a managed service. Users provision Kyma clusters via the BTP Cockpit.                                                            |
| **Converter Configuration** | A JSON file loaded by KIM at startup (and reloaded on change) that defines provider-specific defaults: Kubernetes versions, machine image versions, networking CIDR ranges, and audit log backend mappings.          |
| **CRD**                     | Custom Resource Definition — a Kubernetes extension mechanism that defines a new API object type. KIM defines the `Runtime` and `GardenerCluster` CRDs.                                                            |
| **FSM**                     | Finite State Machine — the orchestration pattern used by the Runtime Controller. Each step of the provisioning workflow is a `stateFn` function. The FSM loop calls state functions in sequence until done or requeued. |
| **Garden cluster**          | The Kubernetes cluster that hosts the Gardener API server and the `Shoot` CRs for a given Gardener project. KIM accesses the Garden cluster using a dedicated kubeconfig.                                           |
| **Gardener**                | An open-source Kubernetes-native system for managing the full lifecycle of Kubernetes clusters across multiple cloud providers. KIM uses Gardener's `Shoot` API as its cluster provisioning backend.                 |
| **GardenerCluster**         | A KIM-defined CRD (legacy) used to represent a managed cluster for the purpose of kubeconfig rotation. Managed by the GardenerCluster Controller.                                                                  |
| **KCP**                     | Kyma Control Plane — the cluster on which KIM runs. `Runtime` and `GardenerCluster` CRs are created in the `kcp-system` namespace on this cluster.                                                                 |
| **KEB**                     | Kyma Environment Broker — the component responsible for creating, updating, and deleting `Runtime` CRs in response to user actions in BTP. KEB is the primary consumer of KIM's API.                                |
| **KIM**                     | Kyma Infrastructure Manager — this system. A Kubernetes operator that manages the lifecycle of Gardener-based Kubernetes clusters for the Kyma environment.                                                         |
| **Kubeconfig**              | A YAML file containing connection credentials (server URL, CA certificate, client token) for a Kubernetes cluster. KIM generates time-limited kubeconfigs for SKR clusters and stores them as Secrets in `kcp-system`. |
| **Kyma**                    | SAP's open-source cloud-native application runtime, built on Kubernetes. KIM provisions the Kubernetes clusters on which Kyma modules run.                                                                          |
| **kyma-system**             | The namespace created on every SKR cluster by KIM as part of post-provisioning configuration. Kyma modules are installed into this namespace.                                                                        |
| **OIDC**                    | OpenID Connect — an identity layer on top of OAuth 2.0. KIM configures the Kubernetes API server on each SKR to trust one or more OIDC providers, enabling federated authentication.                                |
| **Registry Cache**          | A Gardener extension (`registry-cache`) that caches container images on the cluster nodes to reduce pull latency and bandwidth. KIM configures this extension when a `RegistryCacheConfig` CR is found on the SKR.   |
| **RegistryCacheConfig**     | A CRD defined by the Kyma registry-cache module and deployed on SKR clusters. Users create instances of this CRD to request registry caching. KIM reads these CRs and propagates the configuration to Gardener.     |
| **Runtime**                 | The primary CRD managed by KIM. A `Runtime` CR represents a single Kyma cluster and contains the full desired state for provisioning (provider, region, workers, networking, OIDC, security, cache configuration).   |
| **Runtime Bootstrapper**    | An optional component installed by KIM on SKR clusters after provisioning. It installs base Kyma components required for module management.                                                                          |
| **Runtime Watcher**         | A component (external to KIM) that watches SKR clusters for CR changes and emits events to KIM. Planned for use in the registry cache v2 architecture to replace polling.                                           |
| **Shoot**                   | The Gardener CRD representing a managed Kubernetes cluster. KIM creates and manages `Shoot` resources in the Garden cluster. The term "Shoot" comes from Gardener's botanical naming convention.                     |
| **SKR**                     | Shoot Kubernetes Runtime — the Kubernetes cluster provisioned by Gardener for a given `Shoot` CR. This is the cluster that end users interact with; it runs their applications and Kyma modules.                    |
| **stateFn**                 | A Go function type (`func(context.Context, *fsm, *systemState) (stateFn, *ctrl.Result, error)`) that represents a single step in the Runtime Controller's Finite State Machine.                                     |
| **systemState**             | The data structure passed between FSM state functions. It holds the current `Runtime` CR instance and any intermediate data accumulated during the reconciliation cycle.                                              |
