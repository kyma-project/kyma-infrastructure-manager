Risks and Technical Debts
=========================

| Priority | Risk / Technical Debt                                   | Description                                                                                                                                                                                                                                  | Suggested Mitigation                                                                                                             |
|----------|---------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------|
| High     | Registry Cache Controller uses polling instead of push  | The Registry Cache Config Controller periodically polls SKR clusters for `RegistryCacheConfig` changes (ADR 002, v1 architecture). This introduces latency between the user creating a cache config and KIM applying it to Gardener.          | Integrate the Runtime Watcher (planned in ADR 003) to receive push notifications from SKR, eliminating the polling lag.          |
| High     | GardenerCluster CR is a legacy resource                 | The `GardenerCluster` CR and its controller exist for historical reasons (pre-dating the `Runtime` CR). It creates a second code path for kubeconfig rotation that must be maintained in parallel with the Runtime CR lifecycle.               | Migrate kubeconfig rotation into the Runtime FSM as a dedicated state step and deprecate the `GardenerCluster` CR.               |
| Medium   | ADR 003 (Registry Cache v2) is in Proposed status       | The refined registry cache architecture (RegistryCacheConfig CRD, validating webhook, Runtime Watcher integration, credential sync) is not yet implemented. The gap between the accepted ADR 002 and the proposed ADR 003 may cause confusion.  | Accept or reject ADR 003 explicitly. If accepted, implement and close ADR 002 as superseded.                                     |
| Medium   | FSM state chain is implicit / linear                    | The sequence of FSM states is encoded as a chain of function return values. There is no declarative state transition table or graph, making it non-trivial to visualise the full state machine or detect unreachable states statically.        | Introduce a declarative state transition registry or generate a state machine diagram from the code as part of CI.               |
| Medium   | No graceful degradation when Gardener is unreachable    | When the Gardener API server is temporarily unavailable, all pending reconciliations are requeued with exponential backoff. During extended outages, the requeue queue may grow large, and there is no circuit breaker to avoid thundering herds.| Implement a Gardener availability check at the start of each reconcile cycle and skip reconciliation when Gardener is known to be down, resuming once it recovers. |
| Medium   | Dynamic SKR clients are created per reconciliation      | A new Kubernetes client is constructed for each SKR interaction. This adds latency and may cause connection exhaustion during high-throughput provisioning bursts.                                                                             | Introduce a client cache with TTL-based eviction keyed by Runtime UID, reusing clients across reconcile cycles for the same SKR. |
| Low      | Converter configuration reload race condition           | The Config Reload Watcher updates the in-memory configuration when the file changes. If a reconcile cycle reads the configuration concurrently with a reload, it may observe a partially-updated configuration.                               | Protect the configuration with a read-write mutex; ensure the entire configuration object is swapped atomically.                  |
| Low      | End-to-end tests depend on a live Gardener environment  | E2E tests in `test/e2e/` require a real Gardener project and cloud provider credentials. They cannot be run in standard CI without a dedicated environment, reducing test coverage in pull request pipelines.                                  | Introduce an integration test harness with a Gardener stub or use the Gardener local provider for CI-friendly E2E tests.         |
| Low      | `hack/runtime-migrator` is not covered by tests         | The migration tooling in `hack/runtime-migrator/` is critical for data migrations but has limited automated test coverage. A bug in migration tooling can corrupt Runtime CR state across many clusters.                                      | Add unit and integration tests for migration scenarios; require manual review of all migration scripts before execution.          |
