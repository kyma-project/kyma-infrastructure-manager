Runtime View
============

This section describes the most architecturally significant runtime scenarios.

Scenario 1: Initial Cluster Provisioning
-----------------------------------------

Triggered when KEB creates a new `Runtime` CR in the `kcp-system` namespace.

```
KEB                KCP API Server        Runtime Controller (FSM)        Gardener API
 │                       │                         │                          │
 │ create Runtime CR     │                         │                          │
 │──────────────────────►│                         │                          │
 │                       │ reconcile event         │                          │
 │                       │────────────────────────►│                          │
 │                       │                 sFnTakeSnapshot                    │
 │                       │                 sFnInitialise                      │
 │                       │                 (validate labels, set Pending)     │
 │                       │                 sFnSelectShootProcessing            │
 │                       │                 (Shoot does not exist → create)    │
 │                       │                         │ create Shoot              │
 │                       │                         │──────────────────────────►
 │                       │                 sFnWaitingShootCreation             │
 │                       │                 (requeue every 60s)                │
 │                       │                         │◄── Shoot Ready ──────────│
 │                       │                 sFnPatchShoot                      │
 │                       │                 (audit log, OIDC, ACL, net filter) │
 │                       │                         │ patch Shoot               │
 │                       │                         │──────────────────────────►
 │                       │                 sFnWaitingForShootReconcile         │
 │                       │                 (requeue every 30s)                │
 │                       │                         │◄── Shoot Reconciled ─────│
 │                       │                 sFnConfigureSKR                    │
 │                       │                 (kubeconfig secret, OIDC CMs)      │
 │                       │                 sFnCreateKymaNamespace              │
 │                       │                 sFnApplyClusterRoleBindings         │
 │                       │                 [sFnPrepareRegistryCache]           │
 │                       │                 [sFnFinalizeRegistryCache]          │
 │                       │                 [sFnInitializeRuntimeBootstrapper]  │
 │                       │                 sFnUpdateStatus (state=Ready)       │
 │                       │◄────────────────────────│                          │
 │ observe status=Ready  │                         │                          │
 │◄──────────────────────│                         │                          │
```

Notable aspects:
- The FSM loop runs synchronously within a single `Reconcile()` call where possible; steps that require waiting on Gardener return a `ctrl.Result{RequeueAfter: ...}` so the controller releases the worker thread.
- The `sFnPatchShoot` step merges KIM-managed fields onto the Shoot without overwriting fields managed by Gardener.
- Optional steps (registry cache, runtime bootstrapper) are only entered when the corresponding feature flags are enabled and the Runtime CR contains the relevant configuration.

Scenario 2: Cluster Update (Shoot Patch)
-----------------------------------------

Triggered when KEB updates an existing `Runtime` CR (e.g., new Kubernetes version, changed OIDC config).

```
KEB                KCP API Server        Runtime Controller (FSM)        Gardener API
 │                       │                         │                          │
 │ update Runtime CR     │                         │                          │
 │──────────────────────►│                         │                          │
 │                       │ reconcile event         │                          │
 │                       │────────────────────────►│                          │
 │                       │                 sFnTakeSnapshot                    │
 │                       │                 sFnInitialise                      │
 │                       │                 sFnSelectShootProcessing            │
 │                       │                 (Shoot exists → patch)             │
 │                       │                 sFnPatchShoot                      │
 │                       │                         │ patch Shoot               │
 │                       │                         │──────────────────────────►
 │                       │                 sFnWaitingForShootReconcile         │
 │                       │                         │◄── Shoot Reconciled ─────│
 │                       │                 sFnConfigureSKR / ClusterRoleBindings│
 │                       │                 sFnUpdateStatus (state=Ready)       │
 │                       │◄────────────────────────│                          │
```

Scenario 3: Cluster Deletion
------------------------------

Triggered when KEB sets a `deletionTimestamp` on the `Runtime` CR.

```
KEB                KCP API Server        Runtime Controller (FSM)        Gardener API
 │                       │                         │                          │
 │ delete Runtime CR     │                         │                          │
 │──────────────────────►│                         │                          │
 │                       │ reconcile event         │                          │
 │                       │────────────────────────►│                          │
 │                       │                 sFnTakeSnapshot                    │
 │                       │                 sFnSelectShootProcessing            │
 │                       │                 (deletionTimestamp set → delete)   │
 │                       │                 sFnDeleteShoot                     │
 │                       │                 (set deletion confirmation          │
 │                       │                  annotation; state=Terminating)    │
 │                       │                         │ annotate Shoot            │
 │                       │                         │──────────────────────────►
 │                       │                 (requeue every 90s until gone)     │
 │                       │                         │◄── Shoot deleted ─────── │
 │                       │                 sFnDeleteKubeconfig                │
 │                       │                 (remove kubeconfig Secret)         │
 │                       │                 sFnUpdateStatus                    │
 │                       │                 (remove finalizer)                 │
 │                       │◄────────────────────────│                          │
```

Scenario 4: Kubeconfig Rotation
--------------------------------

Handled independently by the GardenerCluster Controller.

```
GardenerCluster CR    GardenerCluster Controller     Gardener API     KCP Secret Store
        │                       │                         │                  │
        │ reconcile event       │                         │                  │
        │──────────────────────►│                         │                  │
        │                       │ check kubeconfig age    │                  │
        │                       │ (age > minRatio * expiry)                  │
        │                       │ request new kubeconfig  │                  │
        │                       │────────────────────────►│                  │
        │                       │◄─ new kubeconfig ───────│                  │
        │                       │ write kubeconfig Secret │                  │
        │                       │────────────────────────────────────────────►
        │                       │ update GardenerCluster status              │
```
