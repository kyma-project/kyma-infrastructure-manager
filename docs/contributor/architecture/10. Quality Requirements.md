Quality Requirements
====================

Quality Tree
------------

```
Quality
├── Reliability
│   ├── Idempotency of reconciliation (QS-R1)
│   ├── Convergence under transient errors (QS-R2)
│   └── Safe concurrent reconciliation (QS-R3)
├── Observability
│   ├── Status accuracy (QS-O1)
│   ├── Metrics completeness (QS-O2)
│   └── Log traceability (QS-O3)
├── Security
│   ├── Kubeconfig rotation (QS-S1)
│   ├── Credential isolation (QS-S2)
│   └── Audit log enforcement (QS-S3)
├── Maintainability
│   ├── FSM extensibility (QS-M1)
│   └── API stability (QS-M2)
└── Performance
    └── Provisioning throughput (QS-P1)
```

Quality Scenarios
-----------------

### Reliability

**QS-R1 — Idempotency of reconciliation**
- **Stimulus:** The Runtime Controller reconciles the same `Runtime` CR twice in quick succession (e.g., due to a spurious watch event after a status update).
- **Response:** The second reconciliation detects that all desired state has already been applied and completes without making any API calls to Gardener or the SKR.
- **Measure:** No duplicate Shoots are created; no unnecessary patch requests are sent to Gardener.

**QS-R2 — Convergence under transient errors**
- **Stimulus:** The Gardener API returns a 429 (rate-limited) or 503 (temporary unavailability) response during shoot creation.
- **Response:** The FSM returns an error; `controller-runtime` requeues the event with exponential backoff. On the next reconcile, the FSM re-enters `sFnWaitingShootCreation` and successfully polls until the Shoot is ready.
- **Measure:** The cluster reaches `Ready` state within the configured retry window; no permanent `Failed` state is set for transient errors.

**QS-R3 — Safe concurrent reconciliation**
- **Stimulus:** 25 `Runtime` CRs are created simultaneously (the default worker count).
- **Response:** Each `Runtime` CR is reconciled independently in its own goroutine. No shared mutable state exists between FSM instances.
- **Measure:** All 25 clusters are provisioned successfully; no data races are detected.

### Observability

**QS-O1 — Status accuracy**
- **Stimulus:** The Gardener Shoot creation fails due to an invalid machine type.
- **Response:** Within one reconcile cycle, `status.state` is set to `Failed`, a `Provisioned` condition is added with reason `GardenerErr`, and a Kubernetes Event is recorded on the `Runtime` CR.
- **Measure:** `kubectl describe runtime <name>` shows the error reason and message without requiring log inspection.

**QS-O2 — Metrics completeness**
- **Stimulus:** A Runtime transitions from `Pending` to `Ready`.
- **Response:** The Prometheus metric `infrastructure_manager_runtime_state` is updated to reflect the new state within the same reconcile cycle.
- **Measure:** Grafana dashboards and alerting rules based on these metrics accurately reflect cluster health without a lag greater than one reconcile cycle.

**QS-O3 — Log traceability**
- **Stimulus:** A support engineer is investigating a provisioning failure.
- **Response:** Structured log entries for the affected `Runtime` CR contain at minimum the runtime name, namespace, current FSM state, and the error message, making it possible to reconstruct the sequence of state transitions from log output alone.
- **Measure:** The root cause of any provisioning failure can be identified from log output without requiring access to the cluster.

### Security

**QS-S1 — Kubeconfig rotation**
- **Stimulus:** A kubeconfig has been in use for `minimal-rotation-time * expiration-time` (default: ~14.4 hours with a 24h expiry and 0.6 ratio).
- **Response:** The GardenerCluster Controller detects the age threshold and requests a new kubeconfig from Gardener, replacing the existing Secret.
- **Measure:** No kubeconfig older than `expiration-time` is active; rotation completes before the existing kubeconfig expires.

**QS-S2 — Credential isolation**
- **Stimulus:** A registry cache is configured with credentials stored in an SKR Secret.
- **Response:** The Runtime Controller reads the credential Secret from SKR and writes it to the Gardener project namespace. The credential is never written to the KCP `kcp-system` namespace.
- **Measure:** Secrets containing registry credentials are present only in the Gardener project namespace; absent from KCP.

**QS-S3 — Audit log enforcement**
- **Stimulus:** KEB creates a `Runtime` CR for a tenant whose ID has no entry in the audit log configuration.
- **Response:** With `--audit-log-mandatory=true`, the FSM sets `status.state = Failed` and `status.conditions` with reason `AuditLogErr`. Provisioning does not proceed.
- **Measure:** No Shoot is created in Gardener for the Runtime CR; the error is visible in status and logs.

### Maintainability

**QS-M1 — FSM extensibility**
- **Stimulus:** A developer needs to add a new post-provisioning step (e.g., installing a new SKR component).
- **Response:** A new `sFn` function is created in `internal/controller/runtime/fsm/` with the standard `stateFn` signature. It is inserted into the state chain by having the preceding state return it as the next state.
- **Measure:** No existing state functions need to be modified; the new step is independently testable with a mock `systemState`.

**QS-M2 — API stability**
- **Stimulus:** KEB is updated to pass a new optional field in the `Runtime` CR spec.
- **Response:** The field is additive and backward-compatible. Existing `Runtime` CRs without the field continue to be processed without error; KIM applies a zero-value default for the new field.
- **Measure:** No migration of existing `Runtime` CRs is required; no downtime during the KIM upgrade.

### Performance

**QS-P1 — Provisioning throughput**
- **Stimulus:** A burst of 25 new `Runtime` CRs is created simultaneously.
- **Response:** The Runtime Controller processes all 25 in parallel using the configured worker pool (`--runtime-ctrl-workers-cnt`, default 25). Gardener API calls are rate-limited to 5 QPS to avoid overloading the Gardener API server.
- **Measure:** All 25 clusters reach `Pending` (Shoot created) state within the Gardener rate limit window; no worker starvation occurs.
